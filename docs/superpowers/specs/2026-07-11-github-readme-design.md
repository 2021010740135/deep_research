# GitHub README 设计说明

## 目标

为 DeepResearch 项目编写一份适合 GitHub 首页展示的中文 README，使访问者能够快速理解项目用途、核心能力、技术架构，并能按照仓库中的真实配置完成本地启动。

## 目标读者

- 希望了解多智能体研究工作流的开发者
- 准备在本地运行、调试或扩展项目的贡献者
- 需要接入 DeepResearch HTTP API 的应用开发者

## 内容结构

README 采用开源项目标准结构：

1. 项目名称、中文简介和一句简短定位
2. 核心特性
3. LangGraph 工作流示意图
4. 技术栈
5. 快速开始，包括环境要求、后端与前端启动方式
6. 环境变量和 `config.json` 配置说明
7. REST 与 SSE API 使用示例
8. 本地 RAG 文档及 PDF 导入方法
9. 项目目录结构
10. 测试命令
11. 当前状态、已知限制与后续计划

## 内容原则

- 全文以中文书写，仅保留技术产品名、命令、路径和 API 字段等必要英文。
- 所有功能描述、命令和配置项均以当前仓库代码为依据。
- 不添加仓库中不存在的许可证、线上演示、贡献规范、截图或发布状态。
- 不展示 `.env` 中的真实密钥，只引用 `.env.example` 并提供空值示例。
- 明确区分必需能力与可选基础设施：通义千问密钥为模型调用所需；博查密钥用于网络检索；PostgreSQL、Redis 和 Milvus 按记忆与 RAG 配置选用。

## 工作流表达

使用 GitHub 原生支持的 Mermaid 流程图展示两条执行路径：简单问题由意图路由直接回答；复杂问题进入规划，随后并行执行网络检索和本地 RAG，再进行证据整合、分析、必要时反思补充检索，最终生成报告。

## 验证标准

- 根目录存在 `README.md`，GitHub Markdown 结构完整。
- README 中的路径、脚本名称、端口、接口和配置变量与代码一致。
- 快速开始覆盖 Python 后端、Vue 前端和命令行三种主要使用入口。
- 文档不包含真实凭据，不覆盖用户已有的非相关改动。
